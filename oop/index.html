Class Cast Exception

Given the class & inheritance structure,

public class SoccerPlayer {
   private String name;
  private double age;
  //
  public String getName() {
   return name;
 }

public class MidfieldSoccerPlayer extends SoccerPlayer {
   private int depthPerception;
   //

   public MidfieldSoccerPlayer(int depthPerception,
      String name, double age, String position, String club) {
      super(name, age, position, club);
      this.depthPerception = depthPerception;
      this.tackleEffectiveness = tackleEffectiveness;
   }
   public int getDepthPerception() {
    return depthPerception;
  }

(1) What happens as a result of the following code?

SoccerPlayer midfielder =
new MidfieldSoccerPlayer(2, .75, "Ronan Curtis", 24, "mid-forward", "Portsmouth");
String playerName = midfielder.getName();

(2) How about this code?

int dp = midfielder.getDepthPerception();

(3) Is there an issue with this code?

MidfieldSoccerPlayer player = (MidfieldSoccerPlayer)new SoccerPlayer();

Analysis

In (1), the 4 tenets of Object Oriented Programming are APIE -

  • Abstraction, pulling out the big ideas and forming software templates called classes. Classes are newed up, or instantiated, to form objects.
  • Polymorphism, the ability of different objects to respond in different ways to the same request (method). Overriding methods is how Java supports Polymorphism.
  • Inheritance, reusing code in one class (the super class or parent class) in another class (child class or sub-class). Facilitated by the extends keyword in Java.
  • Encapsulation, subsuming logic and data inside a class, so the user of the class does not have to worry about details that are within the class.

(2) The keyword extends signifies inheritance. In this case, ChocolateChipCookie class inherits from PlainCookie class. PlainCookie is the superclass or parent class, and ChocolatechipCookie is the child class or subclass.

(1) The age attribute of the parent class cannot be directly accessed by the child class, because it is private. Instead, the parent class setter for the age attribute should be called.

setAge(playerAge);

  public void updateMidfieldPlayerAge(double playerAge)   {
   setAge(playerAge);
    }

(1) cookieA.bake() lists the 350o as in the bake method for class, PlainCookie.

cookieB.bake() lists the 340o as in the bake method for ChocolateChipCookie class.

The class of the object is resolved at runtime, so even though cookieB has been declared as an object of class PlainCookie, it receives a ChocolateChipCookie instance.

(2) The bake method is overriden by the subclass, ChocolateChipCookie, this is an example of Polymorphism.

In (a), the constructor for PlainCookie is called, so PlainCookie is printed.

In (b) and (c), the constructor for ChocolateChipCookie implicitely calls the constructor for PlainCookie first. We see PlainCookie followed by ChocolateChipCookie.

(1) This code uses the SoccerPlayer class to declare an object, and subsequently news up an instance of the MidfieldSoccerPlayer class. This is viable, because the parent class can be used to declare a reference to a child class. The issue is that only the parent class methods are accessible! In this case, "Ronan Curtis" is assigned to variable, playerName

(2) This will not get past the compiler, the method is not visible. The player object was declared as an instance of class SoccerPlayer, and only has access to the methods in SoccerPlayer class, not MidfieldSoccerPlayer.

This can be resolved by using a class cast,

int dp = ((MidfieldSoccerPlayer)midfielder).getDepthPerception();

(3) A Class Cast Exception results. A cast from MidfieldSoccerPlayer to SoccerPlayer is OK, because MidfieldSoccerPlayer is a SoccerPlayer. However, MidfieldSoccerPlayer is a specialized SoccerPlayer, you cannot cast from SoccerPlayer instance to MidfieldSoccerPlayer instance - this results in a class cast exception.
