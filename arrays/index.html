Sorting Arrays

An array of instances of class, Employee, has been established with Employee instances, by method populateEmployees,

int[] employees = new int[10];
populateEmployees(employees);
The array is manipulated based on employee number. Employee has attribute number, and accessor, getNumber. Method format manipulates the array based on attribute number


public static int format(Employee[] arr) {
   for(int index = 0; index < arr.length; index++) {
    int pos = index - 1;
     while (pos >= 0 && arr[pos].getNumber() > arr[index].getNumber()) {
       arr[pos + 1] = arr[pos];
       pos = pos - 1;
       }
       arr[pos + 1] = arr[index];
    }
    }

format(employees);

(1) The fourth item printed is 23. The enhanced for loop cycles through each element in sequence, stopping at the last one.

(2) Fizz is printed 6 times. The regular for loop cycles through each item based on the value of the sentry or loop variable, index. All of the odd numbers in the array are printed, like nums[0] is 13, 13 % 2 is 1. Same for the elements with values 1,23,99,11,77.

(3) 4 elements are printed. The while loop variable starts at 2, and each time through the loop it gets incremented by 1, stopping at nums.length - 1, which is 5. The element at index 2 is 1, the elment at index 3 is 23, then 99, and finally the element at index 5 is 11.

This causes an array index out of bounds error. The valid index values for an array are from 0, first element, to the last element, index value of the number of elements in the array, less 1. Here, the number of elements in the array is 3, so the highest viable index is 2.

(1) The first element is newed up and appropriately accessed, to set the attribute, name, to "Harry Kane" .The valid index values for an array are from 0, first element, to the last element, index value of the number of elements in the array, less 1.

An attempt is made to set the last element's name to "Troy Parrott", but this results in an access violation as that index position in the array does not have an instance of class SoccerPlayer. When an array of objects is newed up, the elements all default to the null reference, null.

(2) All the player names are printed. We are told that the array has been populated with valid instances, and that the attributes have been set.

(1) The array is sorted in ascending order. Method format implements an insert sort algorithm, using the array passed in, arr. The code assumes a sorted part of the list, and that the first element is in this sorted part. Each subsequent item is examined and inserted into the correct position in the sorted part of the list.

(2) 1105, 1203, 1304, 4501, 1337

The sort algorithm shown in (2) is selection sort in ascending order, in which the array is processed sequentially, with the smallest item in the rest of the list swapped with the current item, if smaller.
